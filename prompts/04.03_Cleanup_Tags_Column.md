# Cleanup Task: Remove Deprecated `tags` Column from `MoodEntry`

## Context
We have successfully refactored the tagging system to use a many-to-many relationship (`MoodEntryTagCrossRef`). The original `tags: List<String>` column in the `MoodEntry` table is now redundant and should be removed to keep the database schema clean and avoid confusion.

## Goal
Remove the `tags` property from the `MoodEntry` entity and the corresponding column from the database.

## Steps

### 1. Update `MoodEntry` Entity
*   File: `app/src/main/java/net/maiatoday/moodsnap/data/MoodEntry.kt`
*   Action: Remove `val tags: List<String>`.

### 2. Update `MoodEntryDao`
*   File: `app/src/main/java/net/maiatoday/moodsnap/data/MoodEntryDao.kt`
*   Action: Ensure no queries or inserts reference the old `tags` column. (Most should be fine as Room handles the entity structure, but double-check custom queries if any).

### 3. Update Database Version & Migration
*   File: `app/src/main/java/net/maiatoday/moodsnap/data/AppDatabase.kt`
*   Action:
    *   Increment `version` to 3.
    *   Add a migration strategy.
        *   *Option A (Development)*: `.fallbackToDestructiveMigration()` is already enabled, so this will automatically wipe and recreate the table without the column.
        *   *Option B (Production)*: Write a `Migration` (2 -> 3) that:
            1. Creates a new table `mood_entries_new` without the `tags` column.
            2. Copies data from `mood_entries` to `mood_entries_new`.
            3. Drops `mood_entries`.
            4. Renames `mood_entries_new` to `mood_entries`.

### 4. Fix Compilation Errors
*   Files:
    *   `AddEditViewModel.kt`: Update `MoodEntry` constructor calls to remove the `tags` argument.
    *   `Converters.kt`: Remove unused converters for `List<String>` if they are no longer used elsewhere.
    *   Any test files that instantiate `MoodEntry`.

## Verification
*   Build and run the app.
*   Verify that adding/viewing mood entries still works correctly with the new relationship-based tags.
