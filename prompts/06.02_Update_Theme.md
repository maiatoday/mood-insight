Role: Senior Android Developer
Task: Synchronize the application's Material 3 theme with new design specifications.

Context:
The project is a Jetpack Compose Android app. New design assets (colors and typography) have been provided by the design team in the .agent/resources/06-design-sync/material-theme/ directory.

Objective:
Update the application's theme to match the new design while ensuring that functional "custom colors" (e.g., those used for specific mood levels or status indicators) are preserved.

Workflow:
1. Research & Comparison:
       * Analyze the theme files in .agent/resources/06-design-sync/material-theme/ui/theme/ (specifically Color.kt, Type.kt, and Theme.kt).
       * Compare them with the current implementation in app/src/main/java/net/maiatoday/moodsnap/ui/theme/.
2. Identify Protected Values:
       * Locate any color definitions in the current Color.kt that are not part of the standard Material 3 ColorScheme (e.g., specific semantic colors for moods). Document these as "protected."
3. Implementation:
       * Update Color.kt with the new hex values from the resource folder, keeping the protected values intact.
       * Update Type.kt with the new font family and text style configurations.
       * Update Theme.kt to ensure the lightColorScheme and darkColorScheme correctly map the new values.
4. Verification & Deployment:
       * Perform a clean build of the project using ./gradlew assembleDebug.
       * Deploy the updated application to the running emulator and verify that the UI reflects the new theme and fonts.

Constraints:
   * Do not delete existing color constants that are actively used in the UI but missing from the new design spec.
   * Maintain Material 3 naming conventions.
